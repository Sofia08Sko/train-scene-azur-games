# Train Miner — тестовое задание для Azur Games

Ниже можно найти краткое описание выполненной работы, список реализованных пунктов тестового задания, структуру проекта и инструкцию по запуску сцены в Unity.

Проект выполнен для компании **Azur Games** в качестве **тестового задания** на позицию **3D Generalist Unity/Cocos**.

## Версия Unity
**Unity 2021.3.27f1**

## О проекте
В рамках тестового задания была собрана мини-локация амбара в казуальной стилистике для игры **Train Miner**.  
Сцена подготовлена с учетом просмотра в **портретной** и **горизонтальной** ориентациях.

## Как запустить проект
1. Открыть проект в **Unity 2021.3.27f1**.
2. Открыть сцену:

   `Assets/Train_Project/Scenes/Demo.unity`
   
3. Запустить сцену через **Play**.

## Ориентации и камеры
Для проекта были подготовлены **две отдельные камеры** под разные ориентации экрана — портретную и горизонтальную.  
Это было сделано для более точной настройки композиции и кадра, так как добиться нужного визуального результата одной камерой для обеих ориентаций не удалось.

После запуска **Play Mode** между камерами можно переключаться **по нажатию пробела**.

Если переключение по пробелу не сработало сразу:
1. кликните мышкой по окну **Game**;
2. после этого снова нажмите **Space**.

## Важное замечание по окну Game
Из-за технических особенностей Unity ориентация и разрешение окна **Game** в редакторе меняются вручную.

Для корректной проверки двух режимов рекомендуется:
- для **портретного режима** использовать разрешение **1080 × 1920**;
- для **горизонтального режима** использовать разрешение **1920 × 1080**.

После ручной смены разрешения в окне **Game** можно переключить камеру нажатием **Space**.

## Управление в Play Mode
- **Space** — переключение между портретной и горизонтальной камерами

## Что выполнено
В проекте реализовано:

- мини-локация амбара в стилистике casual / low-poly;
- интеграция и настройка поезда в сцене;
- интеграция персонажа в сцену;
- анимация персонажа в состоянии ожидания;
- игровой UI:
  - надпись **Tap to start** сверху по центру;
  - анимированный курсор внизу экрана;
  - счётчик шестерёнок / гаек слева сверху;
  - счётчик монет справа сверху;
- анимации UI:
  - анимация текста **Tap to start**;
  - анимация курсора;
  - ripple-эффект нажатия;
- базовые VFX:
  - клубы дыма из трубы;
  - атмосферная пыль;
  - мягкие солнечные ореолы у окон;
- освещение сцены без использования URP и PostProcess;
- настройка сцены под две ориентации экрана.

## Структура проекта

### `Assets/Train_Project/Animations`
Анимации персонажа и контроллер анимации:
- `Character_Controller.controller`
- FBX-анимации персонажа

### `Assets/Train_Project/Fonts`
Шрифты проекта и TMP Font Assets.

### `Assets/Train_Project/Linghting`
Настройки освещения сцены:
- `LS_MainScene.lighting`

### `Assets/Train_Project/Materials`
Материалы сцены, UI и VFX:
- основной материал окружения;
- материал персонажа;
- материалы дыма, emissive-элементов, окна, skybox.

### `Assets/Train_Project/Models`
Основные 3D-модели проекта:
- амбар;
- поезд;
- персонаж;
- дополнительные элементы окружения.

### `Assets/Train_Project/Prefabs`
Префабы сцены и UI:
- мелкие элементы окружения;
- UI-префаб счётчика.

### `Assets/Train_Project/Scenes`
Сцены проекта и связанные с ними baked lighting data:
- `Demo.unity`
- `Demo/LightingData.asset`
- lightmaps и reflection probe data

### `Assets/Train_Project/Scrips`
Скрипты проекта:
- `CameraOrientationSwitcher.cs`
- `UIOrientationLayout.cs`

### `Assets/Train_Project/Textures`
Основные текстуры проекта:
- color atlas;
- skybox texture;
- текстура окна;
- вспомогательные UI / VFX текстуры.

### `Assets/Train_Project/UI`
UI-ассеты проекта:
- исходные иконки;
- анимации UI;
- контроллеры UI-анимаций.

### `Assets/Train_Project/VFX`
Префабы и меши визуальных эффектов:
- дым;
- пыль;
- ореол от окон;
- меш облака для дыма.

## Использованные решения
- Built-in Render Pipeline
- Без URP
- Без PostProcess
- TextMeshPro для UI
- Отдельная настройка UI под две ориентации экрана
- Baked + mixed/realtime lighting в зависимости от задач сцены
